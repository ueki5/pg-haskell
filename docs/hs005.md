# hs005 — バイトコードコンパイラ＋スタック仮想マシン

## 概要

hs004の式言語にバイトコードコンパイラとスタック仮想マシン（VM）を追加する。
hs004のトリーウォーキングインタープリタ（`eval`）を廃止し、次の2段階に分離する。

```
ソース文字列
  → tokenize → [Token]
  → parse    → Expr (AST)
  → compile  → [Instr]   ← 新規（コードジェネレータ）
  → run      → Int       ← 新規（仮想マシン）
```

hs004の `Compiler` モジュールをベースとして、`compile` と `run` を追加実装する。

## 構成

| ファイル | 役割 |
|---|---|
| `src/Compiler.hs` | hs004から引き継ぎ。`eval` を削除し `compile`・`run` を追加 |
| `app/Main.hs` | CLIエントリポイント。`compile` → `run` のパイプラインに変更 |
| `test/Spec.hs` | hspecユニットテスト |

## 実行方法

```sh
# ソースファイルを指定して実行
hs005 FILE
```

出力は命令列と実行結果の2行（ASTの代わりに命令列を表示）。

```
Instructions: [Push 1,Push 2,Push 3,Push 4,ISub,IMul,IAdd]
-1
```

## 言語仕様

hs004と同じ（変更なし）。

- 値は符号付き整数のみ
- 演算は加(`+`)、減(`-`)、乗(`*`)、除(`/`)のみ
- 括弧で演算の優先度を制御

## ライブラリ (`Compiler`) の設計

### 命令セット型（新規）

スタック機械向けの命令セットを定義する。

```haskell
data Instr
  = Push Int   -- 定数をスタックに積む
  | IAdd       -- スタックトップ2要素を加算して積み直す
  | ISub       -- スタックトップ2要素を減算（下 - 上）して積み直す
  | IMul       -- スタックトップ2要素を乗算して積み直す
  | IDiv       -- スタックトップ2要素を除算（下 ÷ 上）して積み直す
  | INeg       -- スタックトップの符号を反転して積み直す
  deriving (Show, Eq)
```

### コードジェネレータ（新規）

ASTをスタック機械の命令列に変換する。後置順（ポストオーダー）でASTを走査する。

```haskell
compile :: Expr -> [Instr]
compile (Lit n)   = [Push n]
compile (Add l r) = compile l ++ compile r ++ [IAdd]
compile (Sub l r) = compile l ++ compile r ++ [ISub]
compile (Mul l r) = compile l ++ compile r ++ [IMul]
compile (Div l r) = compile l ++ compile r ++ [IDiv]
compile (Neg e)   = compile e ++ [INeg]
```

例: `1 + 2 * (3 - 4)` → `Add (Lit 1) (Mul (Lit 2) (Sub (Lit 3) (Lit 4)))`

```
[Push 1, Push 2, Push 3, Push 4, ISub, IMul, IAdd]
```

実行トレース:
```
stack: []
Push 1 → [1]
Push 2 → [2, 1]
Push 3 → [3, 2, 1]
Push 4 → [4, 3, 2, 1]
ISub   → [-1, 2, 1]   (3 - 4)
IMul   → [-2, 1]      (2 * -1)
IAdd   → [-1]         (1 + -2)
```

### 仮想マシン（新規）

命令列をスタックを用いて逐次実行する。

```haskell
run :: [Instr] -> Either String Int
```

- スタックは `[Int]` で表現（先頭がトップ）
- 命令列が空になったとき、スタックに要素が1つだけ残れば成功
- 二項演算はスタックから2要素をポップし、結果をプッシュ
- `IDiv` でゼロ除算を検知して `Left "division by zero"`
- スタック不足・実行後の余剰要素はエラー

### 公開関数

```haskell
tokenize :: String -> Either String [Token]
parse    :: [Token] -> Either String Expr
compile  :: Expr -> [Instr]   -- 新規（eval の代替）
run      :: [Instr] -> Either String Int  -- 新規
```

## 依存ライブラリ

- `base` のみ（hs004と変更なし）
- CLIに `optparse-applicative`

## テスト方針

| describe | 主なテスト内容 |
|---|---|
| `compile` | 各ノード型の命令列、複合式の命令順序 |
| `run` | 各命令の実行、ゼロ除算エラー、スタック不足エラー |
| `compile` + `run` | hs004の `eval` テストと同じ入力・出力で結合確認 |
