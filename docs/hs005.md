# hs005 — バイトコードコンパイラ＋スタック仮想マシン

## 概要

hs004の式言語にバイトコードコンパイラとスタック仮想マシン（VM）を追加する。
hs004のトリーウォーキングインタープリタ（`eval`）を廃止し、次の2段階に分離した。

```
ソース文字列
  → tokenize → [Token]
  → parse    → Expr (AST)
  → compile  → [Instr]   ← コードジェネレータ
  → run      → Int       ← 仮想マシン
```

hs004の `Compiler` モジュールをベースとして、`compile` と `run` を追加実装。

## 構成

| ファイル | 役割 |
|---|---|
| `src/Compiler.hs` | 字句解析・構文解析・コードジェネレータ・VMの全ロジック（ライブラリ） |
| `app/Main.hs` | CLIエントリポイント |
| `test/Spec.hs` | hspecユニットテスト |

## 実行方法

```sh
# ソースファイルを指定して実行
hs005 FILE

# ヘルプ表示
hs005 --help
```

ファイルパスは必須引数。出力は命令列と実行結果の2行。

```
Instructions: [Push 1,Push 2,Push 3,Push 4,ISub,IMul,IAdd]
-1
```

## 言語仕様

hs004と同じ（変更なし）。

- 値は符号付き整数のみ
- 演算は加(`+`)、減(`-`)、乗(`*`)、除(`/`)のみ
- 括弧で演算の優先度を制御
- 言語機能は式のみ（文・ステートメントなし）

## ライブラリ (`Compiler`) の設計

### トークン型・AST型

hs004から変更なし。

```haskell
data Token = TInt Int | TPlus | TMinus | TStar | TSlash | TLParen | TRParen
  deriving (Show, Eq)

data Expr
  = Lit Int
  | Add Expr Expr
  | Sub Expr Expr
  | Mul Expr Expr
  | Div Expr Expr
  | Neg Expr
  deriving (Show, Eq)
```

### 命令セット型

スタック機械向けの命令セット。

```haskell
data Instr
  = Push Int
  | IAdd
  | ISub
  | IMul
  | IDiv
  | INeg
  deriving (Show, Eq)
```

| 命令 | 動作 |
|---|---|
| `Push n` | 定数 `n` をスタックに積む |
| `IAdd` | トップ2要素をポップし、加算結果をプッシュ |
| `ISub` | トップ2要素をポップし、`下 - 上` の結果をプッシュ |
| `IMul` | トップ2要素をポップし、乗算結果をプッシュ |
| `IDiv` | トップ2要素をポップし、`下 ÷ 上` の結果をプッシュ |
| `INeg` | トップ要素をポップし、符号反転してプッシュ |

### 公開関数

```haskell
tokenize :: String -> Either String [Token]
parse    :: [Token] -> Either String Expr
compile  :: Expr -> [Instr]
run      :: [Instr] -> Either String Int
```

### 文法（再帰下降パーサ）

hs004から変更なし。

```
expr   ::= term   (('+' | '-') term)*
term   ::= factor (('*' | '/') factor)*
factor ::= INT | '(' expr ')' | '-' factor
```

### コードジェネレータ

ASTを後置順（ポストオーダー）で走査し、スタック機械向け命令列に変換する。

```haskell
compile :: Expr -> [Instr]
compile (Lit n)   = [Push n]
compile (Add l r) = compile l ++ compile r ++ [IAdd]
compile (Sub l r) = compile l ++ compile r ++ [ISub]
compile (Mul l r) = compile l ++ compile r ++ [IMul]
compile (Div l r) = compile l ++ compile r ++ [IDiv]
compile (Neg e)   = compile e ++ [INeg]
```

例: `1 + 2 * (3 - 4)` のコンパイル結果

```
[Push 1, Push 2, Push 3, Push 4, ISub, IMul, IAdd]
```

実行トレース:

```
       stack
Push 1 → [1]
Push 2 → [2, 1]
Push 3 → [3, 2, 1]
Push 4 → [4, 3, 2, 1]
ISub   → [-1, 2, 1]    (3 - 4)
IMul   → [-2, 1]       (2 * -1)
IAdd   → [-1]          (1 + -2)
```

### 仮想マシン

スタック（`[Int]`、先頭がトップ）を持ち回しながら命令を逐次処理する。

- 命令列を消費し終えたとき、スタックに要素が1つだけ残れば成功
- `IDiv` でゼロ除算を検知して `Left "division by zero"`
- スタック要素が不足する場合は `Left "stack underflow"`
- 実行後にスタックの余剰要素が残る場合は `Left "invalid stack state after execution"`
