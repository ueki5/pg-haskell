# hs006 — x86-64ネイティブコード生成

## 概要

hs005の式言語コンパイラにx86-64アセンブリ生成バックエンドを追加する。
hs005の `[Instr]` を入力とし、Linux x86-64向けアセンブリ（AT&T記法）を出力する。
生成したアセンブリは `gcc` でネイティブバイナリにコンパイルする。

```
ソース文字列
  → tokenize → [Token]
  → parse    → Expr (AST)
  → compile  → [Instr]
  → codegen  → アセンブリ文字列   ← 新規
  → .s ファイル出力 → gcc → 実行ファイル
```

## 構成

| ファイル | 役割 |
|---|---|
| `src/Compiler.hs` | hs005から流用（`tokenize`, `parse`, `compile`, `Instr` 型） |
| `src/CodeGen.hs` | 新規。`[Instr]` → アセンブリ文字列の変換ロジック |
| `app/Main.hs` | CLIエントリポイント。`.s` ファイル出力と `gcc` 呼び出し |
| `test/Spec.hs` | hspecユニットテスト |

## 実行方法

```sh
# ソースファイルを指定してコンパイル（デフォルト出力: ./out）
hs006 FILE

# 出力ファイル名を指定
hs006 FILE -o NAME

# ヘルプ表示
hs006 --help
```

`hs006` は内部で `gcc` を呼び出して実行ファイルを生成する。
実行ファイルを手で実行すると計算結果が標準出力に出る。

```sh
$ hs006 expr.txt -o calc
$ ./calc
-1
```

## 言語仕様

hs005と同じ（変更なし）。

## モジュール設計

### `Compiler` モジュール

hs005から変更なし。公開関数・型は以下の通り。

```haskell
tokenize :: String -> Either String [Token]
parse    :: [Token] -> Either String Expr
compile  :: Expr -> [Instr]
```

```haskell
data Instr = Push Int | IAdd | ISub | IMul | IDiv | INeg
  deriving (Show, Eq)
```

### `CodeGen` モジュール（新規）

```haskell
codegen :: [Instr] -> String
```

`[Instr]` を受け取り、アセンブリテキスト全体を返す。
アセンブラ・リンカは `gcc` に任せるため、モジュール内では文字列生成のみを行う。

## アセンブリ出力仕様

### 対象環境

- OS: Linux
- アーキテクチャ: x86-64
- ABI: System V AMD64
- 記法: AT&T記法（GASフォーマット）

### 出力構造

```asm
    .section .text
    .globl main
main:
    pushq %rbp
    movq  %rsp, %rbp

    # [Instr] から生成した計算コード

    popq  %rsi                  # スタックトップ（計算結果）を取り出す
    leaq  fmt(%rip), %rdi       # フォーマット文字列
    xorl  %eax, %eax
    call  printf

    xorl  %eax, %eax            # 終了コード 0
    popq  %rbp
    ret

    .section .rodata
fmt:
    .string "%d\n"
```

### `Instr` → アセンブリ対応表

| Instr | 生成するアセンブリ |
|---|---|
| `Push n` | `pushq $n` |
| `IAdd` | `popq %rax` / `popq %rbx` / `addq %rbx, %rax` / `pushq %rax` |
| `ISub` | `popq %rax` / `popq %rbx` / `subq %rax, %rbx` / `pushq %rbx` |
| `IMul` | `popq %rax` / `popq %rbx` / `imulq %rbx, %rax` / `pushq %rax` |
| `IDiv` | `popq %rcx` / `cmpq $0, %rcx` / `je .div_zero` / `popq %rax` / `cqto` / `idivq %rcx` / `pushq %rax` |
| `INeg` | `popq %rax` / `negq %rax` / `pushq %rax` |

計算にはハードウェアスタック（`%rsp`）をそのまま使う。
`vm` の `[Int]` スタックと1対1で対応するため、レジスタ割り当ては不要。

### スタックアライメント

System V AMD64 ABI では `call` 命令の直前に `%rsp` が16バイト境界に揃っている必要がある。

- `main` 入口で `pushq %rbp` 後、`%rsp` は16バイト境界に揃う
- 計算後にスタックには値が1つだけ残る（`[Instr]` の正しい実行結果）
- `popq %rsi` でその値を取り出すと `%rsp` が元の境界に戻る
- その状態で `call printf` を実行するため、追加のアライメント調整は不要

### ゼロ除算の処理

x86-64 の `idivq` はゼロ除算で `SIGFPE` を発生させるため、事前に検査する。

```asm
    popq  %rcx
    cmpq  $0, %rcx
    je    .div_zero
    popq  %rax
    cqto
    idivq %rcx
    pushq %rax
    ...

.div_zero:
    leaq  errmsg(%rip), %rdi
    call  puts
    movl  $1, %edi
    call  exit
```

`.div_zero` ラベルは関数内で一意にする必要があるため、
複数の `IDiv` が存在する場合はラベルに連番を付与する（`.div_zero_0`, `.div_zero_1`, ...）。

## `Main` の動作フロー

1. ソースファイルを読み込む
2. `tokenize` → `parse` → `compile` で `[Instr]` を得る
3. `codegen` でアセンブリ文字列を生成する
4. 一時ファイル（`/tmp/hs006_XXXX.s`）に書き出す
5. `gcc 一時ファイル.s -o 出力ファイル` を実行する
6. 一時ファイルを削除する

## 依存ライブラリ

- `base`
- `process`（`gcc` 呼び出しに使用）
- `temporary`（一時ファイル生成に使用）
- CLIに `optparse-applicative`

## テスト方針

| describe | 主なテスト内容 |
|---|---|
| `codegen` | 各 `Instr` が期待するアセンブリ行を含むか（部分文字列マッチ） |
| 結合テスト | `compile` → `codegen` → `gcc` → 実行、結果がhs005の `run` と一致するか |

`codegen` の出力全体をスナップショットテストするのは変更に弱いため、
各命令に対応するアセンブリ命令が含まれているかを個別に検証する方針とする。
結合テストでは `System.Process` で生成バイナリを実行し、標準出力を `run (compile ast)` の結果と比較する。
